# CLAUDE.md — Inside Cupertino

## Project Overview

**Inside Cupertino** is a static editorial site covering the Apple ecosystem, built with Astro 5 and deployed to Cloudflare Pages. It runs two parallel content pipelines:

1. **Aggregator** — Hourly RSS fetch of Apple news headlines (links only, no body content).
2. **Drafter** — Daily AI-assisted article drafting via the Claude API. Every draft is saved with `draft: true` and requires human review and editing before publication.

The editorial model is intentional: avoid Google's spam penalties for unreviewed AI-generated content by enforcing a human-in-the-loop gate before any article goes live.

```bash
npm install

npm run dev            # http://localhost:4321
```

```bash
npm run dev          # Local dev server (shows draft articles)
npm run build        # Production build → dist/ (hides draft articles)
npm run preview      # Preview production build locally
npm run aggregate    # Refresh aggregator.json (no API calls)
npm run draft        # AI-draft articles (requires ANTHROPIC_API_KEY)

```

### Adding an Article Manually
1. Create `src/content/articles/<slug>.md`
2. Include all required frontmatter (see schema below)
3. Set `draft: false` to publish immediately, or `draft: true` to queue for review
4. `npm run build` to validate schema at build time

## Article Frontmatter Schema

Defined in `src/content/config.ts` and enforced by Zod at build time.

```yaml
---
title: "..."           # Required. 10–120 characters.
description: "..."     # Required. 50–200 characters. Used as meta description.
pubDate: 2026-05-01    # Required. ISO 8601.
updatedDate: ...       # Optional. ISO 8601.
draft: false           # Required. Must be `false` to appear in production.
author: "Name"         # Optional. Defaults to "Inside Cupertino".
tags:
  - apple
  - iphone
heroImage: "https://..." # Optional. Used in OG meta and article header.
heroAlt: "..."           # Required if heroImage is set.
source:                  # Optional. For articles sourced from external coverage.
  name: "Publication"
  url: "https://..."
---
```

**Critical:** Never set `draft: false` on AI-generated content without human review and editing first.

## Design System

### Themes
- **Light:** "Morning Gallery" palette
- **Dark:** "Nocturnal Intellectual" palette
- **Storage:** `localStorage('ic-theme')`; falls back to system preference → dark
- **Toggle:** Flip `class="dark"` on `<html>` — theme script runs inline before first paint to prevent flash

### Color Tokens
All colors are CSS custom properties defined in `src/styles/global.css` as space-separated RGB triplets (enables Tailwind alpha support via `/ <opacity>`). Mapped into Tailwind via `tailwind.config.mjs`.

Key tokens: `background`, `surface`, `surface-container`, `surface-container-high`, `primary`, `secondary`, `on-surface`, `outline`, `outline-variant`.

### Typography
- `font-headline` / `font-serif` → Newsreader (editorial headlines, body)
- `font-body` / `font-label` → Manrope (UI labels, nav, captions)

### Key Design Rules
1. **No `1px solid` borders** — Use `bg-surface-container-*` background layers for separation.
2. **Gold gradient** — Primary brand accent. Direction: `135deg` in light, `45deg` in dark.
3. **Drop cap** — Applied to article body (first `<p>` via `.prose-editorial`).
4. **Minimal border-radius** — Default 0.125rem (Material Design 3 style).

### Utility Classes (global.css)
- `.ghost-border` — Subtle `1px solid` using `outline-variant` at 15% opacity
- `.glass-header` — Backdrop blur + semi-transparent surface
- `.gold-gradient` — Brand gradient (buttons, hover states, accents)
- `.prose-editorial` — Opinionated type scale for article body
- `.drop-cap` — 5rem serif first letter

## Content Pipeline Details

### Aggregator (`scripts/aggregate-feeds.mjs`)
- Fetches 5 RSS feeds in parallel: 9to5Mac, MacRumors, The Verge, AppleInsider, Six Colors
- 15-second timeout per feed; failures are logged and skipped
- Takes up to 12 items per feed, sorted by `pubDate` descending
- Caps total at 50 items (override with `--max N`)
- Writes `src/data/aggregator.json` with `generatedAt`, `sources`, and `items`
- **No API calls, no cost** — safe to run frequently

### Drafter (`scripts/draft-from-feeds.mjs`)
- Reads `scripts/.drafted-urls.json` to skip already-drafted URLs
- Calls the Claude API to write original analysis (not a rewrite) for each new story
- Generates slug: `<title-kebab-case>-<8-char-hash>.md`
- All output articles have `draft: true` — human must review before publish
- Updates and commits `.drafted-urls.json` (this file is committed to Git)
- Optionally fetches hero images from Unsplash if `UNSPLASH_ACCESS_KEY` is set

## CI/CD (GitHub Actions)

Two cron jobs in `.github/workflows/scheduled.yml`:

| Job | Schedule | What it does |
|-----|----------|-------------|
| `aggregate` | Hourly at :15 | Refresh `aggregator.json`, commit if changed |
| `draft` | Daily at 13:00 UTC | Generate new draft articles, commit if changed |

Both jobs:
- Support manual trigger via `workflow_dispatch`
- Run with 3 retry attempts + exponential backoff
- Use `git pull --rebase` before push to handle concurrent commits
- Append `[skip ci]` to commit messages to avoid triggering Cloudflare rebuilds
- Run on Node 20

Bot commits use identity `inside-cupertino-bot`.

## Key Conventions

- **Components are zero-JS by default.** Astro islands (`client:*`) are not used. All interactivity is either CSS or inline `<script>` tags.
- **Theme toggle is a single inline script** in `BaseLayout.astro`, run before first paint to prevent flash of wrong theme.
- **`isProd` flag** controls draft visibility: `const isProd = import.meta.env.PROD`.
- **All images use `loading="lazy"`** — no eager loading.
- **Tailwind classes only** — no inline `style=""` attributes except for dynamic values.
- **ES modules throughout** — `"type": "module"` in `package.json`; all scripts use `.mjs` extension.
- **No database, no API routes** — pure static site.

## Common Tasks for AI Assistants

### Adding a new page
1. Create `src/pages/<name>.astro`
2. Import `BaseLayout` and wrap content in it
3. Pass `title`, `description`, and optionally `heroImage` as props to `BaseLayout`

### Modifying the article schema
1. Edit `src/content/config.ts`
2. Update the Zod schema
3. Update any existing `.md` frontmatter that references the changed field
4. Run `npm run build` to validate

### Adding a new color token
1. Add CSS variable to both `:root` and `.dark` blocks in `src/styles/global.css` as RGB triplets
2. Map it in `tailwind.config.mjs` under `theme.extend.colors`

### Changing RSS feed sources
- Edit the `FEEDS` array in both `scripts/aggregate-feeds.mjs` and `scripts/draft-from-feeds.mjs`

### Running the draft script locally
```bash
ANTHROPIC_API_KEY=sk-ant-... npm run draft
# or
ANTHROPIC_API_KEY=sk-ant-... npm run draft:dry
```

## What Not to Do

- **Do not set `draft: false` on AI-generated content without reviewing it.** Editorial integrity is core to the project.
- **Do not add SSR or API routes.** The site is intentionally pure static.
- **Do not use `1px solid` borders.** Use surface-container backgrounds for separation.
- **Do not use `client:*` directives** unless absolutely necessary — the site is zero-JS by default.
- **Do not commit `.env`** — it is gitignored and may contain API keys.
- **Do not delete `scripts/.drafted-urls.json`** — it is the deduplication state for the draft pipeline and is intentionally committed.
