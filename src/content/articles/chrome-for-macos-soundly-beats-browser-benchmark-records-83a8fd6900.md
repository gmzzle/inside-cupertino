[AppleInsider](https://appleinsider.com/articles/26/06/06/chrome-for-macos-soundly-beats-browser-benchmark-records?utm_source=rss) reported that Chrome just set new browser benchmark records on macOS, and the real story here is that Google did it on an M5 MacBook Pro. Apple's hardware, Google's win.

This is awkward for Apple. Safari is supposed to be the native advantage browser - tightly integrated with macOS, optimized for Apple Silicon, first in line for hardware acceleration features. And yet here's Chrome, a cross-platform browser built primarily for Windows and Linux volume, smoking the competition on Apple's own chips. Google's blog post claims these are records "across all browsers," which very much includes Safari.

The specific benchmarks matter less than the trend. Chrome has been getting faster on macOS while Safari has mostly been getting... careful. Apple's browser updates lately focus on privacy features, tab groups, and passkey integration. All good things, but none of them make websites render faster or JavaScript execute quicker. Meanwhile Google spent the last year hammering on performance optimizations, and it shows.

This isn't just a bragging rights problem. Developers notice when Chrome runs their web apps better on Macs than Safari does. That calculus influences which browser they test against, which APIs they prioritize, and whether they bother filing WebKit bug reports. If Safari can't win performance tests on its home turf, it cedes mindshare to Chrome in ways that compound over time.

Apple's likely response will be that benchmarks don't reflect real-world usage, which is sometimes true and sometimes cope. Either way, Google just posted a highlight reel using Apple's fastest hardware, and Safari wasn't in the frame.
